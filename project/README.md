## Ok, it’s my toys — статический каталог

Готовый к использованию проект на чистом HTML, CSS и JavaScript. Повторяет макеты mockup-1/2/3 и открывается напрямую через `index.html`.

### Структура

```
project/
  index.html
  css/
    styles.css       # базовая стилистика, сетка и карточки
    modal.css        # оверлеи, модалки, карусель
    responsive.css   # брейкпоинты (1200 / 768)
  js/
    app.js           # загрузка товаров, рендер, модалка товара
    modal.js         # менеджер модалок и focus-trap
    carousel.js      # переключение фото в карточке товара
    dataLoader.js    # fetch JSON с товарами
  data/
    products.json    # 10 примеров товаров
  assets/
    products/        # изображения, используемые в каталоге
    icons/           # иконки доставки, QR-коды
    mockups/         # положите сюда mockup-1/2/3 для хранения
```

### Запуск

1. Скачайте папку `project/`.
2. Откройте `index.html` двойным кликом или перетащите в окно браузера.
3. Всё взаимодействие (модалки, карусель, загрузка данных) работает без сервера.

### Настройка контента

- **Товары**: отредактируйте `data/products.json`. Можно добавлять или удалять поля, главное — сохранить структуру и корректные пути к изображениям.
- **Изображения**: замените файлы в `assets/products/`. Пути в JSON должны совпадать с именами файлов.
- **Модальные тексты**: обновите соответствующие блоки в `index.html`.
- **Цвета и отступы**: хранятся в `css/styles.css` через CSS-переменные.

### Брейкпоинты

- ≥1200 px — 4 колонки каталога.
- 768–1199 px — 2 колонки.
- <768 px — 1 колонка, модалки растягиваются почти на весь экран.

### Дополнительно

- Фокус-трап, закрытие по ESC и клику вне модалки реализованы в `modal.js`.
- Анимации (hover карточек, появление оверлея и модалок) находятся в CSS.
- Для обновления мокапов сохраните файлы `mockup-1.png`, `mockup-2.png`, `mockup-3.png` в `assets/mockups/` — это упростит дальнейшее ревью дизайна.
